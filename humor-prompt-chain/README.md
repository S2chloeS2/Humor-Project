# Humor Prompt Chain Tool

A Next.js admin tool for managing humor flavor prompt chains and generating captions via the AlmostCrackd pipeline API.

## What It Does

This tool lets superadmins build and test **humor flavors** ‚Äî ordered sequences of LLM prompt steps that transform an input image into funny captions.

**Example flavor chain:**
1. Describe the image in plain text
2. Find something absurd or ironic about the description
3. Output five short, punchy captions based on that angle

---

## Features

- **Humor Flavor CRUD** ‚Äî create, edit, delete flavors with a slug and description
- **Step Management** ‚Äî add/edit/delete steps per flavor with system prompt, user prompt, and temperature
- **Step Reordering** ‚Äî move steps up/down to change the chain order
- **Test a Flavor** ‚Äî paste an image URL and call the live pipeline API to see generated captions
- **View Captions** ‚Äî browse all captions previously generated by a flavor
- **Dark / Light / System theme** ‚Äî toggle in the sidebar, persists across sessions
- **Auth-gated** ‚Äî only `is_superadmin` or `is_matrix_admin` profiles can access

## Getting Started

### 1. Install dependencies

```bash
npm install
```

### 2. Set up environment variables

Copy `.env.local.example` to `.env.local` and fill in your Supabase credentials:

```bash
cp .env.local.example .env.local
```

```env
NEXT_PUBLIC_SUPABASE_URL=your_supabase_url
NEXT_PUBLIC_SUPABASE_ANON_KEY=your_supabase_anon_key
SUPABASE_SERVICE_ROLE_KEY=your_service_role_key
```

### 3. Run dev server

```bash
npm run dev
```

Open [http://localhost:3000](http://localhost:3000) ‚Äî you'll be redirected to login.

---

## Access Control

Login is via **Google OAuth**. After authenticating, the app checks your `profiles` row in Supabase:

```sql
profiles.is_superadmin = true
OR
profiles.is_matrix_admin = true
```

Anyone without these flags is immediately signed out and redirected back to the login page.

## Database Tables Used

|---|---|
| `profiles` | Auth check (`is_superadmin`, `is_matrix_admin`) |
| `humor_flavors` | Flavor list (`id`, `slug`, `description`) |

## Deployment (Vercel)

1. Push to GitHub
2. Import the repo in Vercel, set root directory to `humor-prompt-chain`
3. Add the three environment variables in Vercel project settings
4. Add your Vercel deployment URL to Supabase Auth ‚Üí Redirect URLs:
   ```
   https://your-app.vercel.app/auth/callback
   ```
5. Turn off **Deployment Protection** in Vercel (Settings ‚Üí Deployment Protection ‚Üí Off) so the app is accessible in incognito
